# 2026-02-28 - v0.3.0
- Introduced topology-driven architecture baseline: `CellId`, `IGridTopology`, `GridTopology2D`, and `MazeRuntimeContext`.
- Migrated solver algorithms (BFS/DFS/A*/Dijkstra/Greedy) to neighbor-iteration flow; removed fixed 2D delta coupling.
- Migrated generation algorithms (DFS/Prim/Kruskal/GrowingTree) to topology-driven flow; kept Recursive Division as 2D-only.
- Upgraded config/runtime orchestration: supports `dimension/depth/start_z/end_z` parsing and validation; 3D path is intentionally reserved (not executed).
- Split build targets into `maze_core`, `maze_infra`, `maze_cli`, and completed staged source/header migration from `src/` to `apps/` + `libs/`.
- Unified include/export rules with `BUILD_INTERFACE/INSTALL_INTERFACE`; added package install/export (`MazeGeneratorTargets`, `MazeGeneratorConfig.cmake`).
- Added package-consumer smoke sample (`tests/package_consumer`) validating `find_package(MazeGenerator CONFIG)` and linkability.
- Expanded test coverage for runtime/config/CLI behavior and kept build + ctest + install-consumer smoke green during migration.
